3 Grid-to-Ods Backend

 (require nocell/ods/ods) package: nocell
The nocell/ods/ods library provides utilities to create spreadsheet files from a grid program. Currently the library supports fods (opens in LibreOffice) and ods format (opens in both LibreOffice and Excel). Conversion of sexp to xml is implemented by sxml.

procedure

(sxml->ods sxml-program #:type type) → bytes?

  sxml-program : sxml-program?
  type : (or/c 'fods 'ods)

If type is 'fods, produces a bytes string (see also bytes?) corresponding to the spreadsheet in the "flat" ODS file format. If type is 'ods, produces an "extended" ODS file (a zipped folder).

An ods file is a zipped folder with a .ods extension, containing with four files. First, there is an uncompressed "mimetype" file. Then there are three compressed files: "content.xml", "styles.xml", and "META-INF/manifest.xml". A fods file essentially contains the mimetype, content, and styles information wrapped into a single xml file.

Serialization from sexp to xml is achieved using srl:sxml->xml.

WARNING: system zip is used because file/zip does not support leaving the first file in the archive uncompressed.

procedure

(grid-program->sxml program) → sxml-program?

  program : program?

Translates a nocell/grid/grid program into SXML.

procedure

(bytes->file bstr fn) → exact-nonnegative-integer?

  bstr : bytes?
  fn : string?

Like write-bytes, but also takes a filename. The provided filename should have an extension that matches the spreadsheet format of bstr (".xml" for fods, ".ods" for ods).
